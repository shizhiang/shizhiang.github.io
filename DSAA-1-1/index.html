读书►数据结构与算法分析

引论

在许多问题中，一个重要的观念是：写出一个可以工作的程序并不够。

数学知识复习

指数

Xᴬ*Xᴮ = Xᴬ⁺ᴮ
Xᴬ/Xᴮ = Xᴬ⁻ᴮ
(Xᴬ)ᴮ = Xᴬᴮ
Xᴺ+Xᴺ = 2Xᴺ != X²ᴺ
2ᴺ + 2ᴺ = 2ᴺ⁺¹

对数

在计算机科学中，除非有特殊声明，所有的对数都是以2为底的。
定义：Xᴬ = B，当且仅当logⅹB = A.
定理1.1 log𝖠B = logcB / logcA; C > 0
定理1.2 logAB = logA + logB
其他一些类似定理：logA/B = logA - logB
log(Aᴮ)=BlogA
logX < X(对所有的X>0成立)
log1 = 0, log2 = 1, log1024 = 10, log1048576 = 20

级数

公式1: $\sum_{i=1}^N2^i = {2^{N+1} - 1}$

公式2: $\sum_{i=0}^NA^i = {A^{N+1} - 1 \over A-1}$

模运算

如果N整除A-B，那么我们就说A与B模N同余，记为A≡B(mod N)。
若A≡B(mod N)，则A + C≡B + C(mod N)以及AD≡BD(mod N)。

证明方法

证明数据结构分析中的结论的两个最常用的方法是归纳法和反证法。

归纳法证明

  1. 证明基准情形，就是确定定理对于某个（某些）小的值的正确性。
  2. 进行归纳假设。假设定理对直到某个有限数k的所有的情况都是成立的。然后使用这个假设证明定理对下一个值（k+1）也是成立的。
  3. 定理得证。

定理1.3 如果N≥1，则$\sum_{i=1}^Ni^2 = {N(N+1)(N+2) \over 6}$

反证法证明

反证法证明是通过假设定理不成立，然后证明该假设导致某个已知的性质不成立，从而证明原假设是错误的。

递归简论

当编写递归例程时，关键是要牢记递归的四条基本法则：

  1. 基准情形。必须总有某些基准情形，它无须递归就能解出。
  2. 不断推进。对于那些需要递归求解的情形，每一次递归调用都必须要使求解状况朝接近基准情形的方向推进。
  3. 设计法则。假设所有的递归调用都能运行。
  4. 合成效益法则。在求解一个问题的同一实例时，切勿在不同的递归调用中做重复性的工作。
